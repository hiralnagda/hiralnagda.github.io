Experience

ADP LLC

DevOps/SRE Intern

I identified the challenges faced by the team. There were many tasks which were manually carried out by engineers and consumed large amount of their time. I added value to the team by developing and designing following utilities:

1. WFN System Reliability Dashboard

  • Used HTML, CSS, Bootstrap and JavaScript to create Site Reliability Dashboard.
  • All the reports and self serve utilities are placed at once place in a categorical manner.
  • The user just needs to select the service required for the required environment and hit the view button and they will be directed to the service.

2. GETS CMDB Products Tag Viewer

  • First implemented the program in python to get familiarized with CMDB API to create static GETS CMDB Product Tag Report using requests library.
  • For the final product: technologies used are HTML, CSS, JavaScript, AJAX and jQuery (Datatable)
  • Asynchronous AJAX calls are made to CMDB API to get the number of hosts for each tag.

3. ServiceDesk – Incidents from Yesterday

  • Technologies used: Python, Zeep (Python SOAP client), ServiceDesk Web Service API.
  • Three calls are made to the Service Desk API: Fist to login, one to get all the tickets and another to get assignee names from the assignee code.

4. Kafka Automation

  • The project was first implemented using Fabric and Ansible Tower REST API.
  • For the final implementation, automation is performed using Fabric and kafka-python REST API.
  • This is a flask application that runs with Docker using webserver Nginx.
  • The Docker image installs Nginx, uWSGI (Web Server Gateway Interface) and Flask with all of the necessary configuration bundled in.

5. Program to TAG hosts

  • The python script collects a list of hosts and tags from text files as input.
  • Initially, a POST request is made to CMDB API to get an authentication header token to perform future POST calls. A GET request is made to CMDB API to get host ID from the hostname.
  • To tag the hosts with the required tags. A POST request is made and the token is passed in this call. In response, we get the status of the operation.

6. Server Compliance

  • Host data is pulled from CMDB API for parent tag 190 and corresponding tag ID for the environment.
  • For each report the data from CMDB API is collected in chunks of 200 hosts, in order the make the program efficient.
  • Technologies used: CMDB API, Python, jQuery (Datatables), HTML, CSS, JavaScript and shell scripting.
